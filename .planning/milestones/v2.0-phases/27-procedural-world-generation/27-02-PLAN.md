  <action>
Add the `generate_world_region` procedure to `spacetimedb/src/index.ts`, following the established `generate_creation_content` procedure pattern.

The procedure:

1. **Phase 1 (withTx - read context):**
   - Find WorldGenState row by genStateId. Verify step is PENDING. Update step to GENERATING.
   - Read character data: find character by genState.characterId, get race/class/archetype from CharacterCreationState (by_player index with genState.playerId).
   - Read source region name from genState.sourceRegionId.
   - Build neighbor context using `buildRegionContext()` from `../helpers/world_gen`.
   - Read API key from `llm_config.id.find(1n)`.
   - Check budget via `checkBudget()`.
   - Return all context data needed for the LLM call.

2. **Phase 2 (http.fetch - LLM call):**
   - Build system prompt: `buildWorldGenPrompt('')` (context is embedded in user prompt for world gen).
   - Build user prompt: `buildRegionGenerationUserPrompt(race, class, archetype, sourceRegionName, neighbors)`.
   - Call Claude API using `buildAnthropicRequest('claude-haiku-4-5', systemPrompt, userPrompt)` with `response_format: { type: 'json_object' }` in the request body.
   - Use Haiku per Phase 26 discovery (Sonnet HTTP fails from SpacetimeDB runtime).

3. **Phase 3 (withTx - write results):**
   - Re-read WorldGenState row. Verify step is still GENERATING (guard against stale state).
   - Parse JSON response. Use brace extraction fallback (find first `{` to last `}`) per Phase 26 hardening pattern.
   - Validate parsed JSON has required fields (regionName, locations array with length >= 1).
   - Call `writeGeneratedRegion(tx, parsed, genState)` to write all content.
   - Update WorldGenState: step = COMPLETE, generatedRegionId = region.id.
   - Emit ripple announcement: `appendWorldEvent(tx, 'world', pickRippleMessage(sourceRegionName, parsed.biome, tx.timestamp.microsSinceUnixEpoch))`.
   - Emit personal discovery narrative: `appendPrivateEvent(tx, genState.characterId, character.ownerUserId, 'system', pickDiscoveryMessage(parsed.regionName, tx.timestamp.microsSinceUnixEpoch))`.
   - Increment budget.

4. **Error handling:**
   - Wrap the entire procedure in try/catch.
   - On error: withTx to update WorldGenState step = ERROR, errorMessage = error message.
   - Re-read genState in error withTx to avoid stale state.

5. **Wire procedure into V2 export auto-collection:**
   - Follow the same `_wrapMethod('procedure', ...)` pattern used by `call_llm` and `generate_creation_content`.
