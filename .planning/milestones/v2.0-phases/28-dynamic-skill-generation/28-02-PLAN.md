Add a JSON schema that the LLM must follow for skill generation. The schema asks for an array of 3 skills:
```json
{
  "skills": [
    {
      "name": "string — creative, unique name",
      "description": "string — sardonic System narrator description, 1-2 sentences",
      "kind": "one of: damage, heal, dot, hot, buff, debuff, shield, taunt, aoe_damage, aoe_heal, summon, cc, drain, execute, utility",
      "targetRule": "one of: single_enemy, single_ally, self, all_enemies, all_allies, all_party, lowest_hp_ally, lowest_hp_enemy",
      "resourceType": "one of: mana, stamina, hp, none",
      "resourceCost": "number — resource cost to use",
      "castSeconds": "number — cast time in seconds (0 for instant)",
      "cooldownSeconds": "number — cooldown in seconds",
      "scaling": "one of: str, dex, int, wis, cha, hybrid, none",
      "value1": "number — primary power value (damage, heal amount, etc.)",
      "value2": "number or null — secondary value (DoT ticks, drain heal%, etc.)",
      "damageType": "one of: physical, arcane, divine, nature, fire, ice, shadow, none — required for damage-dealing kinds",
      "effectType": "string or null — effect type for buff/debuff/dot/hot kinds",
      "effectMagnitude": "number or null — effect strength",
      "effectDuration": "number or null — effect duration in seconds"
    }
  ]
}
```

**2. Add buildSkillGenSystemPrompt() to llm_prompts.ts:**
System prompt establishes The System narrator voice:
- "You are The System — a sardonic, omniscient narrator who finds mortals' power struggles amusing"
- "Generate 3 unique abilities for this character. Each should feel like it belongs to THEIR specific journey — not generic fantasy abilities."
- "At least 2 of the 3 must be DIFFERENT kinds (e.g., don't offer 3 damage abilities)"
- "Names should be creative and specific to the character's identity, not generic (no 'Fireball', 'Ice Bolt')"
- "Descriptions should be 1-2 sentences of sardonic commentary from The System"
- Include the JSON schema
- Include the complete list of valid enum values for each field (from mechanical_vocabulary)

**3. Add buildSkillGenUserPrompt(character, race, className, archetype, level, existingAbilities) to llm_prompts.ts:**
User prompt provides character context:

- List of existing ability names and kinds (to avoid duplicates)
- "Generate 3 abilities appropriate for level {level}. Make them distinct from existing abilities."

- Delete any existing PendingSkill rows for this character (in case of retry)
- Insert 3 PendingSkill rows with createdAt = ctx.timestamp, levelRequired = level

- Inserts LlmTask with domain `'skill_gen'`, model `'gpt-5-mini'`, maxTokens 1500n

**2. Add skill_gen domain handler in submit_llm_result (index.ts:560+):**

In the existing submit_llm_result reducer, add a new domain branch:

For error case (`!success`):
- Append a narrative event: `'The System flickers. "Your potential eludes crystallization. The power will come... eventually."'`
- No state revert needed — player can retry via the level-up prompt

For success case:
- Parse contextJson to get characterId
- Call `parseSkillGenResult(resultText, characterId, level)` from skill_gen.ts
- If parse fails, append error event and return
- Call `insertPendingSkills(ctx, characterId, parsedSkills, level)`
- Call `incrementBudget(ctx, ctx.sender)` for cost tracking
- Append narrative event with The System presenting the 3 skills:

Format the presentation as:
```
The System regards you with something resembling interest.

"Level {level}. How quaint. The universe has deigned to offer you three new ways to embarrass yourself:"

[Skill1Name] — {description}
  {kind} | {resourceCost} {resourceType} | {cooldownSeconds}s cooldown | {value1} power

[Skill2Name] — {description}
  {kind} | {resourceCost} {resourceType} | {cooldownSeconds}s cooldown | {value1} power

[Skill3Name] — {description}
  {kind} | {resourceCost} {resourceType} | {cooldownSeconds}s cooldown | {value1} power

"Choose wisely. Or don't. The rejected skills will dissolve into the void, never to return."
```

Use `[SkillName]` brackets so the existing keyword click system works.

**3. Add choose_skill reducer to index.ts:**

Create reducer `choose_skill` with params `{ pendingSkillId: t.u64() }`:
- Find the PendingSkill row by id
- Validate ctx.sender owns the character (look up character by pendingSkill.characterId)
- Get all PendingSkill rows for this character
- Insert the chosen skill into ability_template (copy all fields, set isGenerated: true)
- Get the inserted row's id
- Auto-assign to first available hotbar slot: find existing hotbar slots for character, assign to next empty slot (0-5). If no empty slot, assign to slot 0 with a warning event.
- Delete ALL PendingSkill rows for this character (chosen + unchosen — gone forever)
- Append narrative: `'The System nods. "{SkillName} it is. The others scatter like forgotten dreams. You will never see them again."'`
- Append a system event showing the skill was learned: `"You learned [{SkillName}] — {kind}, {value1} power, {cooldownSeconds}s cooldown"`
