**Rewrite useCombat.ts** to work with the round-based engine:

1. **Add subscriptions** for new tables:
   - CombatRound (via useTable)
   - CombatAction (via useTable)
   - CombatNarrative (via useTable)

2. **Replace real-time combat state** with round-based computed properties:
   - `currentRound`: computed from CombatRound rows filtered by active combatId, find the one with state !== 'resolved' (or the highest roundNumber)
   - `roundState`: computed from currentRound.state ('action_select', 'resolving', 'resolved')
   - `myAction`: computed from CombatAction rows -- find action for my character in current round
   - `hasSubmittedAction`: computed boolean from myAction existence
   - `roundTimeRemaining`: computed ref that updates via requestAnimationFrame -- calculates seconds remaining from currentRound.timerExpiresAtMicros vs server clock offset (reuse existing server clock offset pattern from the codebase if available, otherwise use client timestamp)

3. **Action submission functions:**
   - `submitAction(actionType, abilityTemplateId?, targetEnemyId?, targetCharacterId?)`: calls conn.reducers.submitCombatAction with characterId + args
   - `submitAbility(abilityTemplateId, targetEnemyId)`: convenience wrapper
   - `submitAutoAttack(targetEnemyId)`: convenience wrapper
   - `submitFlee()`: convenience wrapper with actionType 'flee'

4. **Combat initiation** -- modify pull/attack flow:
   - Keep startPull functionality (clicking enemy -> "Approach carefully or charge in?" prompt)
   - The prompt should be shown in the narrative stream as a system message with clickable options
   - After pull resolves, round-based combat begins (server creates first CombatRound)

5. **Action prompt building** -- new computed property `actionPromptMessage`:
   - Only populated when roundState === 'action_select' and !hasSubmittedAction
   - Builds a formatted message string with:
     - Timer countdown: "Choose your action ({N}s remaining):"
     - List of hotbar abilities as [AbilityName] (manaCost mana) with cooldown status
     - [Auto-attack] option
     - [Flee] option
     - [More abilities...] option for non-hotbarred abilities
     - Target list: all living enemies as clickable names
   - Uses existing [bracket] keyword syntax so clickNpcKeyword handler processes clicks
   - Format abilities with cooldown info: [Ability Name] if ready, ~~Ability Name~~ (Ns) if on cooldown

6. **Round summary building** -- new computed property `roundSummaryMessage`:
   - After round resolves (state === 'resolved'), build a summary with textual HP bars
   - Format (monospace block):
     ```
     --- Round {N} ---
     [████████████░░░░░░] 67/100 HP  Dire Wolf (L3)
     [████░░░░░░░░░░░░░░] 23/80 HP   Ash Crawler (L2)
     ---
     [██████████████░░░░] 82/120 HP  YourCharacter
     ```
   - HP bar format: 20-character width, filled chars vs empty chars
   - Colors via {{color:HEX}} tags (green #69db7c >50%, yellow #ffd43b 25-50%, red #ff6b6b <25%)
   - Emit this as an event via appendPrivateEvent from the SERVER (plan 01/02 should handle this), OR build client-side from CombatRound + participant HP data

7. **Loot handling in narrative stream:**
   - When CombatResult shows victory and CombatLoot has items:
   - Build a narrative message with clickable loot: "[Take Rusty Sword]" "[Take All Loot]" "[Dismiss]"
   - Wire clickNpcKeyword to call takeLoot/takeAllLoot/dismissCombatResults reducers for those keywords

8. **Remove real-time polling/timers** from useCombat:
   - Remove any setInterval or tick-based state polling
   - All state comes from table subscriptions (reactive via useTable)
   - roundTimeRemaining uses requestAnimationFrame for smooth countdown display

**In useContextActions.ts:**
- Add a check: if the player's character is in active combat (activeCombat exists and state is not completed/defeated), hide the context action bar (return empty actions array or add an `isInCombat` computed ref)
- The NarrativeConsole or App.vue should use this to hide the action bar during combat
