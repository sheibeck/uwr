<objective>
Wire LLM narration into the round-based combat engine. Create narration qualification logic, build the combat round user prompt with event details, handle narration results in submit_llm_result, and integrate the trigger point into resolveRound.

Purpose: This makes combat feel like a story. The LLM narrates key moments (combat intro, dramatic rounds, victory/defeat) while mechanical text handles the tactical information. The sardonic System narrator voice is already defined in buildCombatNarrationPrompt -- this plan wires it into the round lifecycle.

Output: Working LLM narration pipeline for combat: qualifying rounds create LlmTask, client proxy calls LLM, results broadcast as gold/amber narrative text to all combat participants.
</objective>

**Create spacetimedb/src/helpers/combat_narration.ts** with these functions:

1. **RoundEventSummary type:**
```typescript
type RoundEventSummary = {
  combatId: bigint;
  roundNumber: bigint;
  narrativeType: 'intro' | 'round' | 'victory' | 'defeat';
  playerActions: Array<{
    characterName: string;
    actionType: string;
    abilityName?: string;
    targetName?: string;
    damageDealt?: bigint;
    healingDone?: bigint;
    wasCrit?: boolean;
    missed?: boolean;
    fled?: boolean;
    fleeSuccess?: boolean;
  }>;
  enemyActions: Array<{
    enemyName: string;
    abilityName?: string;
    targetName?: string;
    damageDealt?: bigint;
    healingDone?: bigint;
    wasCrit?: boolean;
  }>;
  effectsApplied: string[];
  effectsExpired: string[];
  deaths: string[];          // Names of killed enemies or fallen players
  nearDeathNames: string[];  // Names of participants below 25% HP
  hasCrit: boolean;
  hasKill: boolean;
  hasNearDeath: boolean;
  participantHpSummary: Array<{ name: string; hp: bigint; maxHp: bigint; isEnemy: boolean }>;
};
```

2. **shouldNarrateRound(narrativeType, narrationCount, remainingBudget):**
   - Always narrate 'intro', 'victory', 'defeat' if budget allows (remainingBudget > 0)
   - For 'round' type: return false if narrationCount >= MAX_COMBAT_NARRATIONS (3)
   - For 'round' type: return false if remainingBudget < NARRATION_BUDGET_THRESHOLD (10)
   - For 'round' type: return true only if hasCrit || hasKill || hasNearDeath
   - Import constants from combat_constants.ts

3. **triggerCombatNarration(ctx, combat, round, events: RoundEventSummary):**
   - Call shouldNarrateRound -- return early if false
   - Get all active combat participants
   - Round-robin budget charge: chargeIndex = narrationCount % participantCount
   - Find the character at chargeIndex, get their ownerUserId -> Player -> identity for budget
   - Check budget via checkBudget(ctx, chargedPlayerIdentity) -- return silently if not allowed
   - Increment budget via incrementBudget
   - Build context string from events (character names, enemy names, location, round number)
   - Call buildCombatNarrationPrompt(context) for system prompt
   - Call buildCombatRoundUserPrompt(events) for user prompt
   - Insert LlmTask with domain: 'combat_narration', model: 'gpt-5-mini', maxTokens: 400n
   - contextJson: { combatId, roundNumber, narrativeType, participantCharacterIds }
   - Increment round.narrationCount on the CombatRound

4. **handleCombatNarrationResult(ctx, task, resultText, success):**
   - Parse contextJson for combatId, roundNumber, narrativeType, participantCharacterIds
   - If !success: silently fail (combat continues without narration, no error message needed)
   - Parse resultText as JSON, extract narrative field (with brace extraction fallback per Phase 26 pattern)
   - Insert CombatNarrative row
   - Broadcast to all participant characters via appendPrivateEvent with kind 'combat_narration'
   - For 'intro' type: also send a 'system' kind message "The System settles in to watch." or similar
   - For victory/defeat with no narration (budget gone): send "The System has lost interest in your skirmish." once

**In llm_prompts.ts, add:**

5. **buildCombatRoundUserPrompt(events: RoundEventSummary):** Returns a user prompt string that describes the round's events in structured format for the LLM:
```
Round {N} of combat:
Players acted:
- {characterName} used {abilityName} on {targetName}, dealing {damage} damage {(CRITICAL HIT!) if crit}
- {characterName} auto-attacked {targetName} for {damage} damage
Enemies acted:
- {enemyName} used {abilityName} on {targetName}, dealing {damage} damage
Effects: {effectsApplied}, Expired: {effectsExpired}
Deaths: {deaths}
Survivors: {hpSummary}

Respond with JSON: { "narrative": "your 2-4 sentence narration of this round" }
```

6. **COMBAT_NARRATION_SCHEMA** constant for the response format:
```
{ "narrative": "string -- 2-4 sentences narrating the combat events with sardonic System voice" }
```

7. **buildCombatIntroUserPrompt(enemyNames, playerNames, locationName):** For combat intro narration.

8. **buildCombatOutroUserPrompt(events, isVictory):** For victory/defeat narration.
