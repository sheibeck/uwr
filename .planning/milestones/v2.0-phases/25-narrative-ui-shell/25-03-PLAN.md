**useNarrativeAnimation.ts** -- Typewriter state machine.

Export `useNarrativeAnimation()` function that returns:
```typescript
{
  animationStates: Ref<Map<string, AnimationState>>;
  isAnimating: ComputedRef<boolean>;
  startAnimation: (eventId: string, text: string) => void;
  skipAll: () => void;
}
```

Where:
```typescript
type AnimationState = {
  sentences: string[];
  revealed: number;
  complete: boolean;
};
```

Implementation:
- `animationStates` is a reactive `Map<string, AnimationState>`
- `startAnimation(eventId, text)`: Split text into sentences using `text.match(/[^.!?]+[.!?]+\s*/g) || [text]`. Create entry in map with `revealed: 0, complete: false`. Start revealing via `revealNext(eventId)`.
- `revealNext(eventId)`: Increment `revealed` on the state entry. If `revealed >= sentences.length`, set `complete: true`. Otherwise, `setTimeout(() => revealNext(eventId), 300)` (300ms between sentences). Store the timeout ID so it can be cleared on skip.
- `skipAll()`: For every entry in the map that is not complete, set `revealed = sentences.length` and `complete = true`. Clear any pending timeouts.
- `isAnimating`: computed that returns true if any entry in the map has `complete === false`.
- Safety: if `isAnimating` stays true for more than 10 seconds (watchEffect with timeout), force-complete all animations to prevent permanent input block (as warned in research pitfall #4).

**NarrativeMessage.vue updates:**

Add optional prop: `animationState: AnimationState | undefined`.

When `animationState` is provided and not complete:
- Split the message into the same sentences the animation uses
- Only render sentences up to `animationState.revealed`
- Each sentence wraps in a `<span>` with CSS transition: `opacity 0.4s ease-in`. Newly revealed sentences start at opacity 0 and transition to 1 via a reactive class/style binding.
- When `animationState.complete` is true or undefined, render all text normally (no animation)

The narrative/llm kind styling from Plan 02 (gold, italic, accent border) is already applied. No changes needed for that.

**NarrativeConsole.vue updates:**

1. Import and use `useNarrativeAnimation()` in the component setup.

2. Watch `combinedEvents` for new events with `kind === 'narrative'` (or however LLM events are identified). When a new narrative event appears that wasn't in the previous list, call `startAnimation(eventKey, event.message)` where eventKey is `${event.scope}-${event.id}`.

3. Pass animation state to each NarrativeMessage: `:animation-state="animationStates.get(\`${event.scope}-${event.id}\`)"`.
