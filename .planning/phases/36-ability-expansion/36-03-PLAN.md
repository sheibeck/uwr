    1. In combat.ts resolveAbility, add dispatch cases for each new kind (see research code examples for implementation details):

       - **song/aura**: Apply long-duration buff (effectDuration default 180n) to self. Songs could apply to party members too if in a group. Map to the existing addCharacterEffect. Log "You begin {ability.name}."

       - **fear**: CC debuff on enemy. Requires combatId. Apply 'stun' effect to target enemy for effectDuration turns. Log "{actor.name}'s {ability.name} fills {enemy} with dread."

       - **travel**: Apply travel-related buff to self. No combat restriction. Use addCharacterEffect.

       - **craft_boost/gather_boost**: Apply loot_bonus or similar effect to self. No combat restriction.

       - **food_summon**: Placeholder — log "You conjure sustenance" and apply a buff effect. No item creation needed this phase.

       - **resurrect**: Placeholder — log message only. Full resurrection mechanics deferred.

       - **pet_command**: Placeholder — log message only. Full pet system deferred.

       - This allows LLM to generate debuff abilities with value1=0 that only apply effects

    3. Verify buff handler already works outside combat (research says it does — confirm by checking that the handler doesn't require combatId for self-targeted buffs).

    4. Add test cases to combat.test.ts for the new dispatch paths and pure buff/debuff behavior.
