<objective>
Fix the CREATION_ABILITY_SCHEMA field name mismatch that causes ALL character creation abilities to be stored as kind:'damage' regardless of what the LLM generates.

Purpose: The LLM schema uses field name "effect" (values: none|dot|heal|buff|debuff|stun) but creation.ts reads "chosen.kind" which is undefined, falling back to 'damage'. This means DoTs never tick, HoTs never heal, and buffs/debuffs never apply -- they all just do direct damage. This is the root cause of UAT gaps 1-4.
Output: Aligned schema field names so creation abilities get the correct kind, plus damageType preservation.
</objective>
