Create `src/composables/useContextActions.ts` that exports `ContextAction` type and `useContextActions` function.

```typescript
export type ContextAction = {
  label: string;        // Button text ("Attack", "Go North")
  command: string;      // Text to submit via submit_intent
  category: 'combat' | 'explore' | 'social' | 'ability';
  disabled?: boolean;   // e.g., on cooldown
  detail?: string;      // Tooltip: mana cost, cooldown time
};
```

The composable takes:
```typescript
{
  selectedCharacter: Ref<Character | null>;
  activeCombat: Ref<CombatEncounter | null>;
  connectedLocations: Ref<Location[]>;
  npcsHere: Ref<Npc[]>;
  hotbarDisplay: Ref<HotbarDisplaySlot[]>;
  isCasting: Ref<boolean>;
  canActInCombat: Ref<boolean>;
}
```

Import types from `../module_bindings/types`. Use existing type names (Character, Location, Npc). For HotbarDisplaySlot, check what useHotbar exports and use the same type (or define inline if needed).

Returns `computed<ContextAction[]>` that:

**In combat** (activeCombat is truthy):
- Map each hotbarDisplay slot that has an abilityKey to a ContextAction with:
  - label: slot.name
  - command: `use ${slot.name.toLowerCase()}`
  - category: 'ability'
  - disabled: cooldownRemaining > 0 or isCasting or !canActInCombat or (kind === 'item' and itemCount === 0)
  - detail: cooldownRemaining > 0 ? `${cooldownRemaining}s` : undefined
- Add a "Flee" action: { label: 'Flee', command: 'flee', category: 'combat' }

**Not in combat** (exploration):
- Map each connectedLocation to: { label: loc.name, command: `go ${loc.name.toLowerCase()}`, category: 'explore' }
- Add "Look" action: { label: 'Look', command: 'look', category: 'explore' }
- Map each NPC to: { label: `Talk to ${npc.name}`, command: `hail ${npc.name.toLowerCase()}`, category: 'social' }

Return the actions array. No reactivity beyond the computed -- this is a pure derivation from game state.
