  <files>src/composables/useHotbar.ts, src/components/NarrativeHotbar.vue</files>
  <action>
1. **Extend useHotbar.ts** to support multiple hotbars:

   a. Add computed `hotbarList`: query `tables.hotbar` filtered by character, sorted by sortOrder. Returns array of `{ id, name, isActive, sortOrder }`.

   b. Add computed `activeHotbar`: the hotbar where isActive=true (or first if none active).

   c. **Update `hotbarAssignments`** to filter by activeHotbar's ID using `tables.hotbarSlot` `by_hotbar` index instead of `by_character`.

   d. Add `switchHotbar(name: string)`: call `conn.reducers.switchHotbar({ characterId, hotbarName: name })`.

   e. Add `prevHotbar()` / `nextHotbar()`: find current hotbar in sorted list, switch to adjacent one (wrapping around).

   f. Add `createHotbar(name: string)`: call `conn.reducers.createHotbar({ characterId, name })`.

   g. Export these new functions alongside existing ones.

   h. **Backward compatibility:** If character has no Hotbar rows yet, `hotbarDisplay` should still work (fall back to by_character filter on HotbarSlot for characters who haven't created a hotbar yet, or lazily trigger default hotbar creation).

2. **Create NarrativeHotbar.vue:**

   A fixed-position strip that sits below the NarrativeHud bar. Always visible when a character is selected.

   Layout:
   - Left arrow button (prev hotbar)
   - Hotbar name label (clickable -- emits 'list-hotbars' or types 'hotbars' command)
   - 10 slot buttons in a row, each showing:
     - Slot number (small, top-left)
     - Ability name (abbreviated if long)
     - Cooldown overlay (dark fill from bottom to top, percentage based)
     - Cooldown seconds remaining (if on cooldown)
   - Right arrow button (next hotbar)

   Props: Accept hotbar data from useHotbar (hotbarDisplay, activeHotbar name, hotbarList).
   Events: `use-slot`, `prev-hotbar`, `next-hotbar`.

   Style: Dark background matching NarrativeHud (#12121a / #1a1a2e). Slots ~40px wide, gold border on assigned slots, gray border on empty. Cooldown overlay uses semi-transparent dark fill. Arrow buttons same style as slots but narrower.

   The component should be compact -- a single row about 36-40px tall.
  </action>
