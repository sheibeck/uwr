**Part A: Reducers in items.ts**

1. `create_hotbar` reducer: Takes `{ characterId: t.u64(), name: t.string() }`. Validate character ownership via requireCharacterOwnedBy. Check count of existing hotbars for character (max 10). Deactivate all existing hotbars for character. Create new Hotbar row with isActive=true, sortOrder = count of existing hotbars. Emit system event confirming creation.

2. `switch_hotbar` reducer: Takes `{ characterId: t.u64(), hotbarName: t.string() }`. Find hotbar by name (case-insensitive) for character. Set all character hotbars isActive=false, set target isActive=true. Emit system event.

3. `swap_hotbar_slots` reducer: Takes `{ characterId: t.u64(), slot1: t.u8(), slot2: t.u8() }`. Find active hotbar for character. Find slots on that hotbar matching slot1 and slot2. Swap abilityTemplateId values. Handle cases where one or both slots are empty.

4. **Update existing `set_hotbar_slot` reducer:** Add hotbarId parameter or auto-detect active hotbar. When setting a slot, use the active hotbar's ID. If no active hotbar exists, create a default "main" hotbar first.

5. **Lazy default hotbar creation:** Add a helper `ensureDefaultHotbar(ctx, character)` that creates a default hotbar named "main" if the character has none. Call this from set_hotbar_slot, swap_hotbar_slots, and any hotbar query in intent.ts.

**Part B: Intent commands in intent.ts**

Add a new section AFTER sell commands:

1. `hotbars` -- list all hotbars for character with slot contents. For each hotbar, show name and slots with ability names. Use clickable keywords for hotbar names.

2. `hotbar add <name>` -- call create_hotbar logic inline (or call the reducer's logic).

3. `hotbar <name>` -- switch active hotbar.

4. `hotbar set <slot> <ability>` -- look up ability by name from character's abilities, call set_hotbar_slot logic for active hotbar.

5. `hotbar swap <slot1> <slot2>` -- call swap logic for active hotbar.

6. `hotbar` (bare) -- show active hotbar contents.
