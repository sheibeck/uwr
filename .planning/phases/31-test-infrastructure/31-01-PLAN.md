<task type="auto">
  <name>Task 1: Create shared mock DB utility</name>
  <files>spacetimedb/src/helpers/test-utils.ts</files>
  <action>
Create `spacetimedb/src/helpers/test-utils.ts` with two exported functions:

1. `createMockDb(seed: Record<string, any[]> = {})` -- Proxy-based auto-creation mock DB:
   - Auto-creates tables on first access (no upfront registration needed)
   - Pre-seeding support: pass `{ tableName: [rows...] }` to pre-populate
   - Auto-increment: replace `id === 0n` and `scheduledId === 0n` with sequential bigint IDs per table
   - Track max seeded ID per table so auto-increment continues from highest existing
   - Insert returns the row (with resolved ID), matching SpacetimeDB behavior
   - `.iter()` returns the array (iterable)
   - `._rows()` returns the raw array for test assertions
   - `.id.find(id)`, `.id.update(row)`, `.id.delete(id)` -- primary key by `id`
   - `.identity.find(id)`, `.identity.update(row)`, `.identity.delete(id)` -- primary key by `identity`
   - `.scheduledId.find(id)`, `.scheduledId.delete(id)` -- for scheduled tables
   - Named index support: `by_owner`, `by_location`, `by_character`, `by_from`, `by_to`, `by_combat`, `by_instance`, `by_vendor`, `by_event`, `by_spawn`, `by_name`, `by_player`, `by_score`, `by_group` -- each returns `{ filter(val), find(val), update(row), delete(val) }`
   - INDEX_TO_COLUMN map for known indexes (from RESEARCH.md pattern). Fallback: `by_X` -> column `XId`
   - Unknown properties also return an index accessor (for flexibility)

2. `createMockCtx(opts)` -- Bundles db + timestamp + sender:
   - `opts.seed` -> passed to createMockDb
   - `opts.sender` -> defaults to `{ toHexString: () => 'mock-identity-hex' }`
   - `opts.timestampMicros` -> defaults to `1_000_000_000_000n`
   - Returns `{ db, timestamp: { microsSinceUnixEpoch }, sender }`

Use the implementation from RESEARCH.md as the starting point. Ensure all index names used in the existing two test files are covered (by_location, by_from, by_to, by_character, by_owner).
  </action>
