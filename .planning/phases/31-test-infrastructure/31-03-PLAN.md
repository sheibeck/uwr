Read intent.ts to extract ALL regex patterns and string comparisons used for command dispatch. Then add:

`describe('intent routing patterns')`:

Test each command regex pattern directly (extract the regex from intent.ts and test matching):

- `look` / `l` / `look <target>` -- regex: `/^(?:look|l)(?:\s+(.+))?$/i`
- `go <destination>` -- regex: `/^go\s+(.+)$/`
- `travel to <destination>` / `travel <destination>` -- regex: `/^travel\s+(?:to\s+)?(.+)$/`
- `say <message>` -- regex: `/^say\s+(.+)$/i`
- `whisper <player> <message>` / `tell` / `w` -- regex: `/^(?:whisper|tell|w)\s+(\S+)\s+(.+)$/i`
- `talk <npc>` / `hail` / `speak` / `talk to <npc>` -- regex: `/^(?:talk|hail|speak)\s+(?:to\s+)?(.+)$/i`
- `consider <target>` / `con` -- regex: `/^(?:consider|con)\s+(.+)$/i`
- `attack <target>` / `fight` / `kill` -- regex: `/^(?:attack|fight|kill)\s*(.*)$/`
- `use <ability>` / `cast` -- regex: `/^(?:use|cast)\s+(.+)$/`

For each pattern test:
- Primary command matches and captures correct groups
- Aliases match (e.g., `l` for `look`, `w` for `whisper`)
- Non-matching input returns null
- Capture groups extract the right values (target name, message text, etc.)

Also test string-equality commands (these are likely matched by `lower === 'xxx'` or `lower.startsWith('xxx')`):
- `help` / `h` / `?`
- `inv` / `inventory` / `i`
- `stats`
- `who`
- `quest` / `quests`
- `group`

Read the actual intent.ts dispatch logic to get the complete list of recognized commands. Test that each one matches and non-commands don't.

For string comparisons, test:
```typescript
it('recognizes help aliases', () => {
  for (const cmd of ['help', 'h', '?']) {
    expect(['help', 'h', '?'].includes(cmd.toLowerCase())).toBe(true);
  }
});
```

Keep tests focused on PATTERN MATCHING, not full dispatch (which would require massive deps object). The patterns themselves are the testable contract.
